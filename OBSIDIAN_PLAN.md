### 2.2 obsidian_writer.py 核心邏輯

```python
"""
obsidian_writer.py — 自動生成 Obsidian notes 並 push 到 GitHub

觸發時機：
  1. 每日 18:30（pipeline 跑完後） → Daily note + Trade notes
  2. 每週五 22:30（audit 跑完後） → Weekly review note
  3. 每月 1 日 → Monthly summary note
  4. 手動觸發 POST /obsidian/generate

依賴：
  - GITHUB_TOKEN（GitHub Personal Access Token，repo 權限）
  - GITHUB_REPO（e.g. "AngusRepo/stockvision-brain"）
  - D1 REST API（讀取交易數據）
"""

class ObsidianWriter:
    """
    從 D1 讀取數據 → 填入 Jinja2 模板 → 生成 .md → git push GitHub
    """

    async def generate_daily(self, date: str) -> dict:
        """
        生成 Daily/{date}.md
        數據來源：
          - market_risk（大盤風險）
          - daily_recommendations（pipeline 結果）
          - paper_positions（持倉快照）
          - paper_daily_snapshots（portfolio 績效）
          - paper_orders（今日交易）
          - decision_logs（L2 決策記錄）
        """

    async def generate_trade_note(self, order: dict) -> dict:
        """
        生成 Trades/{SIDE}-{SYMBOL}-{DATE}.md
        數據來源：
          - paper_orders（交易詳情）
          - decision_logs（因子歸因）
          - predictions（ML 投票明細）
          - 若為 SELL → 配對 BUY order 計算持有天數/損益
        """

    async def generate_weekly_review(self, week_start: str) -> dict:
        """
        生成 Audits/Weekly/{YEAR}-W{WEEK}.md
        數據來源：
          - weekly_audit_reports（L1/L2/L3 已由 audit graph 生成）
          - paper_daily_snapshots（週績效）
          - model_health_daily（模型健康）
        """

    async def generate_pipeline_snapshot(self, date: str) -> dict:
        """
        生成 Pipeline/{date}.md
        數據來源：
          - daily_recommendations（25 檔 screener 結果）
          - predictions（ML 信號）
          - pending_buys KV（T2 debate 結果）
          - sector_flow（主題輪動）
        """

    async def update_model_notes(self) -> dict:
        """
        更新 Models/{model_name}.md（10 個模型各一個）
        數據來源：
          - model_health_daily（最新健康指標）
          - model_lifecycle_events（lifecycle 歷史）
        """

    async def sync_progress_md(self, date: str) -> dict:
        """
        自動更新 stockvision-cloudflare-v12/progress.md
        目的：Claude Code 開新 session 時自動讀取，恢復完整 context

        寫入內容（壓縮摘要，< 2KB）：
          - 最新 session 日期
          - Portfolio 狀態（總資產/報酬率/持倉數/MDD/Sharpe）
          - 今日 Pipeline 摘要（screener→ML→debate 各階段數量）
          - 今日交易（BUY/SELL 列表 + 理由一句話）
          - 模型健康警報（退化中的模型）
          - 最新 Optuna 參數版本
          - 待辦/Action Items（從 weekly review 繼承）
          - Deployments 版本（Worker/ML/Controller）

        寫入位置：
          1. Obsidian vault: Current-State.md（人可讀完整版）
          2. 本地專案: stockvision-cloudflare-v12/progress.md（Claude 讀壓縮版）

        觸發：每日 generate_daily() 完成後自動呼叫
        """

    async def generate_risk_report(self) -> dict:
        """
        生成 Risk/MC-PBO-{date}.md
        數據來源：
          - monte_carlo_results
          - pbo_results
          - backtest_results
        """

    async def _push_to_github(self, files: list[dict]) -> bool:
        """
        用 GitHub Contents API 推送檔案
        POST https://api.github.com/repos/{REPO}/contents/{path}
        - 單次可推多個檔案（用 Git Trees API 做 batch commit）
        """
```

### 2.3 GitHub 推送方式

**方案 A：GitHub Contents API（簡單）**
```python
# 單檔推送
PUT https://api.github.com/repos/{owner}/{repo}/contents/{path}
{
  "message": "daily: 2026-04-07",
  "content": base64_content,
  "sha": existing_sha  # 更新時需要
}
```

**方案 B：Git Trees API（批量，推薦）**
```python
# 1. 取最新 commit SHA
GET /repos/{owner}/{repo}/git/ref/heads/main

# 2. 建 tree（多個檔案）
POST /repos/{owner}/{repo}/git/trees
{ "base_tree": base_sha, "tree": [
    {"path": "Daily/2026-04-07.md", "mode": "100644", "content": "..."},
    {"path": "Trades/BUY-2330-2026-04-07.md", "mode": "100644", "content": "..."},
]}

# 3. 建 commit
POST /repos/{owner}/{repo}/git/commits
{ "message": "auto: daily 2026-04-07", "tree": tree_sha, "parents": [parent_sha] }

# 4. 更新 ref
PATCH /repos/{owner}/{repo}/git/refs/heads/main
{ "sha": commit_sha }
```

推薦方案 B — 一次 commit 推所有當日產生的 notes，commit history 乾淨。

### 2.5 自動同步 progress.md（Claude 跨 session 記憶）

每日 `generate_daily()` 完成後，自動呼叫 `sync_progress_md()`：

1. 從當日生成的 Daily note 提取關鍵數據
2. 壓縮成 < 2KB 的 progress.md 格式
3. 用 GitHub API 推送到兩個位置：
   - `stockvision-brain/Current-State.md`（Obsidian 完整版）
   - `stockvision-cloudflare-v12/progress.md`（Claude 壓縮版，同 repo）

progress.md 內容模板：
```markdown
# Progress — StockVision

## Session {date}

### Portfolio
- Total: ${total_value} ({total_return}%)
- Positions: {count} 檔 | Cash: ${cash}
- MDD: {mdd}% | Sharpe(30d): {sharpe}

### Today's Pipeline
- Screener: {screener_count} → ML BUY: {ml_buy} → T2: {t2_count} 掛單
- Trades: {buy_count} BUY / {sell_count} SELL

### Positions
{symbol} {shares}張 cost:{cost} now:{price} {pnl_pct}%
...

### Model Health
- Degraded: {degraded_models or "None"}
- Optuna params version: {version}

### Deployments
- Worker: {commit_hash}
- ML (Modal): {status}
- Controller (Cloud Run): {status}

### 待做
- [ ] {from weekly review action items}
```

## 風險與注意事項

1. **Conflict**：多裝置同時編輯可能 git conflict → Obsidian Git 設定 auto-merge strategy
2. **大小**：純 .md 很小（100MB 撐 10 年），避免嵌入大圖/PDF
